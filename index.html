How this project uses Unsupervised Learning to analyze player efficiency.

Step 1: Data Collection (NBA API) - Python & Pandas

I utilized the nba_api endpoints to fetch career stats for over 4,500 players. To ensure data quality, I normalized stats to a "Per 75 Possessions" basis to compare eras fairly.


        # Fetching player logs
        from nba_api.stats.endpoints import playercareerstats
        
        # Normalizing Data (Snippet)
        def normalize_stats(df):
            df['PTS_PER_75'] = (df['PTS'] / df['POSS']) * 75
            return df
Step 2: The Math (Euclidean Distance) - Scikit-Learn

To find the "Nearest Neighbor," I treat every player as a vector in 12-dimensional space. We calculate the Euclidean distance between the target player vector ($p$) and every other player vector ($q$).

d(p, q) = \sqrt{\sum_{i=1}^{n} (q_i - p_i)^2}

Before calculation, all features are scaled using Standard Scaler ($z = \frac{x - \mu}{\sigma}$) to prevent high-volume stats (Points) from overpowering defensive metrics (Steals).

Step 3: Unsupervised Learning (K-Means) - Machine Learning

I used the Elbow Method to determine that k=4 was the optimal number of clusters for modern NBA archetypes.

  • Cluster 0: Rotation Scorers
  • Cluster 1: Deep Bench
  • Cluster 2: Primary Stars
  • Cluster 3: Big Men
